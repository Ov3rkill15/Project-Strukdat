# Glosarium Istilah Teknis

## Daftar Istilah dalam Dokumentasi Blockchain E-Commerce

---

## A

**Address (Alamat)**
Identifier unik untuk user atau wallet dalam blockchain. Biasanya berupa string panjang yang di-generate dari public key.

**Algorithm (Algoritma)**
Serangkaian instruksi untuk menyelesaikan masalah tertentu.

**API (Application Programming Interface)**
Interface untuk komunikasi antar software components.

**Asymmetric Cryptography (Kriptografi Asimetris)**
Sistem enkripsi yang menggunakan pasangan public key dan private key.

---

## B

**Balance (Saldo)**
Jumlah cryptocurrency atau token yang dimiliki user.

**Binary Tree (Pohon Biner)**
Tree structure di mana setiap node maksimal memiliki 2 children.

**Block**
Unit dasar dalam blockchain yang berisi kumpulan transaksi, timestamp, dan hash.

**Blockchain**
Distributed ledger yang terdiri dari chain of blocks.

**Block Height**
Nomor urut block dalam chain, dimulai dari 0 (genesis block).

**Byzantine Fault Tolerance (BFT)**
Kemampuan sistem untuk berfungsi meskipun ada node yang berperilaku malicious.

---

## C

**Chain**
Sequence of blocks yang saling terhubung melalui cryptographic hashes.

**Chargeback**
Reversal transaksi payment (tidak ada di blockchain).

**Checkpointing**
Menyimpan state blockchain pada titik tertentu untuk recovery.

**Collision (Tabrakan)**
Situasi di mana dua input berbeda menghasilkan hash yang sama.

**Complexity (Kompleksitas)**
Ukuran efisiensi algoritma dalam terms of time atau space.

**Confirmation**
Validasi transaksi dengan inclusion dalam block yang sudah di-mine.

**Consensus**
Agreement antar nodes dalam distributed system.

**Cryptocurrency**
Digital currency yang menggunakan kriptografi untuk security.

---

## D

**DAO (Decentralized Autonomous Organization)**
Organisasi yang dijalankan oleh smart contracts tanpa central authority.

**Decentralization (Desentralisasi)**
Distribusi kontrol dan decision-making across network.

**DeFi (Decentralized Finance)**
Financial services tanpa intermediaries tradisional.

**Difficulty**
Parameter yang menentukan seberapa sulit mining puzzle.

**Digital Signature**
Cryptographic signature untuk membuktikan authenticity.

**Distributed Ledger**
Database yang tersebar di multiple locations/nodes.

**Double Spending**
Mencoba menggunakan cryptocurrency yang sama dua kali.

---

## E

**Encryption (Enkripsi)**
Proses mengubah data menjadi format yang tidak bisa dibaca tanpa key.

**Escrow**
Third-party yang menyimpan funds hingga kondisi terpenuhi.

**Ethereum**
Platform blockchain yang support smart contracts.

---

## F

**Fee (Biaya)**
Pembayaran untuk transaction processing (biasanya ke miner).

**FIFO (First In First Out)**
Prinsip queue di mana item pertama masuk, pertama keluar.

**Fork**
Perubahan dalam blockchain protocol yang membuat split.

**Fraud**
Aktivitas penipuan atau manipulation.

---

## G

**Gas**
Unit yang mengukur computational effort di Ethereum.

**Genesis Block**
Block pertama dalam blockchain.

**GUI (Graphical User Interface)**
Interface visual untuk user interaction.

---

## H

**Hash**
Output fixed-size dari hash function.

**Hash Function**
Function yang mengubah input menjadi fixed-size string.

**Hash Rate**
Kecepatan computing power dalam mining.

**Hash Table**
Data structure untuk fast lookup menggunakan key-value pairs.

**HMAC (Hash-based Message Authentication Code)**
Teknik untuk verify message authenticity.

---

## I

**Immutability (Ketidakberubahan)**
Property di mana data tidak bisa diubah setelah ditambahkan.

**Incentive**
Reward untuk mendorong behavior tertentu (e.g., mining).

**Index**
Position dari element dalam array atau block dalam chain.

**Integrity (Integritas)**
Jaminan bahwa data tidak dimodifikasi unauthorized.

---

## K

**Key Pair**
Pasangan public key dan private key.

**KYC (Know Your Customer)**
Proses verifikasi identitas customer.

---

## L

**Ledger**
Record of all transactions.

**LIFO (Last In First Out)**
Prinsip stack di mana item terakhir masuk, pertama keluar.

**Linked List**
Data structure di mana setiap element link ke element berikutnya.

---

## M

**Merkle Root**
Hash di root dari Merkle tree yang merepresentasikan semua transactions.

**Merkle Tree**
Tree structure untuk efficient verification of data.

**Miner**
Entity yang melakukan mining untuk validate transactions.

**Mining**
Proses solving computational puzzle untuk add block ke chain.

**Mining Reward**
Cryptocurrency yang diterima miner untuk successfully mining block.

**Mutex (Mutual Exclusion)**
Mechanism untuk thread synchronization.

---

## N

**Network**
Kumpulan connected nodes.

**NFT (Non-Fungible Token)**
Unique digital asset yang tidak bisa ditukar 1:1.

**Node**
Computer yang participate dalam blockchain network.

**Nonce (Number used Once)**
Nilai yang diubah-ubah dalam mining untuk find valid hash.

**Non-repudiation**
Jaminan bahwa sender tidak bisa deny sending message.

---

## O

**O(1), O(n), O(log n)**
Notasi Big-O untuk complexity analysis.

**Off-chain**
Transactions atau data yang tidak stored di blockchain.

**On-chain**
Transactions atau data yang stored di blockchain.

**OpenSSL**
Library untuk cryptographic functions.

---

## P

**P2P (Peer-to-Peer)**
Network architecture tanpa central server.

**Pending Transaction**
Transaction yang belum included dalam block.

**Pointer**
Variable yang stores memory address.

**PoS (Proof of Stake)**
Consensus mechanism berdasarkan stake ownership.

**PoW (Proof of Work)**
Consensus mechanism berdasarkan computational work.

**Previous Hash**
Hash dari block sebelumnya dalam chain.

**Priority Queue**
Queue di mana elements memiliki priority.

**Private Key**
Secret key untuk signing transactions.

**Protocol**
Set of rules untuk communication.

**Public Key**
Key yang bisa dibagikan untuk verifying signatures.

---

## Q

**Queue**
Data structure dengan FIFO ordering.

---

## R

**RAII (Resource Acquisition Is Initialization)**
C++ programming idiom untuk resource management.

**Rollback**
Reverting changes ke previous state.

**Root**
Top node dalam tree structure.

---

## S

**Scalability**
Kemampuan sistem untuk handle increased load.

**SHA-256**
Cryptographic hash function yang menghasilkan 256-bit output.

**Sharding**
Splitting blockchain into smaller pieces untuk scalability.

**Sidechain**
Separate blockchain yang connected ke main chain.

**Signature**
Digital signature untuk authenticate transaction.

**Smart Contract**
Self-executing contract dengan terms in code.

**Stack**
Data structure dengan LIFO ordering.

**STL (Standard Template Library)**
C++ library untuk data structures dan algorithms.

**Supply Chain**
Network dari production hingga delivery.

---

## T

**Tamper-evident**
Property di mana modification akan terdeteksi.

**Throughput**
Number of transactions processed per unit time.

**Timestamp**
Record of waktu kejadian event.

**Token**
Digital asset pada blockchain.

**TPS (Transactions Per Second)**
Metric untuk measuring blockchain performance.

**Transaction**
Transfer of value dari satu address ke address lain.

**Transaction Pool (Mempool)**
Collection of pending transactions.

**Transparency**
Property di mana information accessible untuk all parties.

**Tree**
Hierarchical data structure dengan nodes.

---

## U

**UTXO (Unspent Transaction Output)**
Model untuk tracking ownership (used in Bitcoin).

---

## V

**Validation**
Process of verifying correctness.

**Vector**
Dynamic array dalam C++ STL.

**Verification**
Confirming authenticity atau validity.

---

## W

**Wallet**
Software untuk storing dan managing cryptocurrency.

**Web3**
Decentralized internet based on blockchain.
